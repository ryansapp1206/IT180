Project Overview

This presentation outlines the transition from a basic midterm layout into a fully functional web application.

  • Goal: Convert my midterm project into a dynamic, responsive digital resume that can access live data.
  • Professionalism: Designed to serve as a legitimate, real-world portfolio piece to showcase my technical capabilities as a Computer Science student.
  • Architecture: Built entirely from scratch using semantic HTML, modern CSS, and Vanilla JavaScript.

Design & Layout

  • Colors: High-contrast Slate and Blue theme. I used CSS Variables (`:root`) to maintain strict color consistency across all pages.
  • Fonts: Segoe UI and standard sans-serif fallbacks for a modern aesthetic that prioritizes readability.
  • Spacing & UX: Generous line heights (1.6) and padding ensure the site doesn't feel cluttered. Important elements have hover states to provide immediate user feedback.

Navigation Structure

  • Multi-Page Logic: The `index.html` sits at the root, while secondary pages (About, Projects, Gallery, Contact) are organized cleanly in an `/html` folder.
  • In-Page Navigation: The homepage features a secondary navigation menu utilizing smooth-scrolling anchor links (`href="#about-section"`) to jump to specific sections.
  • Accessibility: Semantic heading hierarchy (H1 -> H2 -> H3) allows screen readers to navigate the content logically.

CSS Grid & Flexbox

I utilized both CSS layout modules based on their specific strengths:

  • Flexbox: Used in the header and navigation bars. It perfectly handles horizontal alignment, and spacing out the links evenly.
  • CSS Grid: Used for the hardware image gallery and GitHub repository cards. By using `repeat(auto-fit, minmax(...))`, the grid automatically calculates how many columns can fit on the screen without needing additional media queries.

JavaScript Features

I implemented several interactive modules to improve the site:

  • Theme Toggle: Allows users to switch between light and dark modes, saving their preference to `localStorage`.
  • Dynamic Greeting: Uses the `Date()` object to determine the time of day and updates the homepage H1 appropriately (Morning/Afternoon/Evening).
  • Form Validation: Intercepts contact form submissions to verify standard email formatting before processing.
  • Gallery Lightbox: Generates a full-screen image overlay dynamically using the DOM `createElement` method.

Code Spotlight: State Persistence

This snippet handles the logic for toggling the dark mode class and saving the user's preference locally so it persists across page loads.


// Listen for clicks, toggle class, and save choice to local storage
themeBtn.addEventListener('click', function() {
    const isDark = document.documentElement.classList.toggle('dark-mode');
    localStorage.setItem('theme', isDark ? 'dark' : 'light');
    updateButtonLabel(isDark);
});

Challenges & Solutions

The Challenge: FOUC (Flash of Unstyled Content)

When navigating between pages, the site would briefly flash the default light theme for a split second before the JavaScript loaded and applied the dark theme.

The Solution

I extracted the `localStorage` check from my main `script.js` file and placed it in a small, blocking <script> tag directly inside the HTML <head>. This forces the browser to read the theme preference and apply the dark-mode class before it paints the screen.
