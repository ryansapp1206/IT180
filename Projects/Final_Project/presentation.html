IT180 Final Project Presentation

Presented by Ryan Sapp

Project Overview

  • The Goal: Transform a static, multi-page midterm assignment into a dynamic, responsive digital resume capable of holding real-world data.
  • The Architecture: Built entirely from scratch using semantic HTML5, modern CSS, and Vanilla JavaScript.
  • The Strategy: Designed an "Executive Summary" homepage for quick scanning, with detailed content structured across a modular, multi-page architecture.

Design & Layout

  • Visual Identity: High-contrast Slate and Blue palette with centralized CSS Variables for strict consistency and theme-switching.
  • Header Imagery: Custom "Hero" header featuring original landscape photography from Dolly Sods, West Virginia.
  • User-Friendly Choices: Clean headings, generous line-heights for readability, and clear semantic labels for all navigation elements.

Layout: Flexbox & CSS Grid

Utilized the two primary CSS layout modules for their specific strengths.

  • Flexbox: Powering the navigation menus and header alignment to space elements evenly.
  • CSS Grid: Powering the hardware image gallery to create a responsive, auto-fitting grid.
/* Auto-fitting Grid for the Project Gallery */
.gallery-container {
    display: grid; 
    grid-template-columns: repeat(auto-fit, minmax(280px, 1fr));
    gap: 20px;
}

Responsive Architecture

Implemented media queries to ensure the portfolio remains functional and clean across desktop and mobile devices.

/* Mobile layout triggers at 600px */
@media (max-width: 600px) {
    nav ul {
        flex-direction: column; /* Stack links vertically */
        gap: 10px;
    }
    .profile-img {
        max-width: 200px; /* Scale down image */
    }
}

Navigation & Interlinking

  • Multi-Page Structure: Seamless linking between Home, About, Projects, Gallery, and Contact pages.
  • UX Enhancement: Integrated smooth-scrolling for the anchor links on the Executive Summary homepage.
  • Accessibility: Proper heading hierarchy (H1-H3) and descriptive alt text for all visual assets.

JavaScript Interactivity

  • State Persistence: Reading and writing to localStorage to remember theme preferences across sessions.
  • DOM Generation: A custom Lightbox modal that creates a dynamic full-screen image view when gallery items are clicked.
  • Validation: Intercepting form submissions to validate email formats before processing.

Key Feature: GitHub API Fetch

Implemented an asynchronous fetch request to pull live repository data directly into the UI.

// Asynchronous repo fetch
fetch(`https://api.github.com/users/ryansapp1206/repos?sort=updated`)
    .then(response => response.json())
    .then(data => {
        data.forEach(repo => {
            const card = document.createElement('div');
            card.innerHTML = `<h4>${repo.name}</h4>`;
            githubContainer.appendChild(card);
        });
    });

Challenges & Solutions

The Challenge: Flash of Unstyled Content (FOUC)

The site would flash light mode for a second before the JavaScript applied the dark theme.

The Solution

I moved the theme check into a tiny blocking script in the HTML <head> to ensure the correct class is applied before the browser paints the screen.

Key Takeaways

  • Scalability: Centralizing styling with CSS Variables makes global changes (like theming) significantly faster.
  • Modern APIs: Reinforced the importance of using Fetch for live data and Web Storage for local state.
  • Semantic Foundation: Learned that clean HTML structure is the prerequisite for implementing complex CSS layouts like Grid.
